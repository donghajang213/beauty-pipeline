# docs/ — 프로젝트 기록

> **무엇**: 코드로 표현되지 않는 지식을 기록하는 폴더.
> **왜**: 기술 선택의 이유와 문제 해결 과정이 곧 면접 답변이 되고, 미래의 나를 위한 기록이 된다. (RULES.md 1장)

## 구조

| 하위 폴더/파일 | 용도 | 작성 시점 |
|---|---|---|
| `setup/` | 수동으로 수행한 부트스트랩 작업 기록 (계정 생성, 결제 연결 등) | 수동 작업 직후 |
| `adr/` | 아키텍처 결정 기록 (`NNN-제목.md`) | 중요한 기술 선택 시 |
| `troubleshooting/` | 30분 이상 헤맨 문제의 증상→원인→해결→배운 점 | 문제 해결 직후 |
| `weekly/` | 주간 회고 (한 일 / 배운 것 / 다음 주 / 비용) | 매주 일요일 |
| `cost-log.md` | 주간 비용 확인 기록 | 매주 일요일 |
| `iam.md` | 서비스 계정별 부여 권한과 이유 | 권한 부여 시 |
