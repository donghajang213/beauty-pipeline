# writing/ — 취업용 글쓰기 (자소서 · 포트폴리오 · 블로그)

> **무엇**: 이 프로젝트를 "글"로 파는 공간. 코드가 실력이라면, 글은 그 실력을 채용담당자에게 전달하는 통로다.
> **왜 저장소 안에**: 프로젝트가 진행될 때마다 소재(트러블슈팅, 결정)를 바로바로 여기에 쌓아야
> 나중에 자소서 쓸 때 "뭐 했더라" 하며 기억을 쥐어짜지 않는다.

## 구조

| 폴더/파일 | 용도 | 쓰는 시점 |
|---|---|---|
| `blog/_template.md` | 티스토리 포스트 양식 | 새 글 쓸 때 복사 |
| `blog/NNN-*.md` | 발행할 글 초안 — 티스토리에 복붙 | 주요 작업 마무리마다 |
| `resume/자소서_소재은행.md` | 자소서 문항에 꽂을 경험 목록 (STAR 형식) | 사건이 생길 때마다 즉시 추가 |
| `resume/포트폴리오_소개.md` | 이력서 첨부/면접용 1페이지 프로젝트 요약 | Phase 끝날 때마다 갱신 |

## ⚠️ 저장소 공개 전 체크

- 자소서에는 개인 정보(지원 회사명, 신상)가 들어갈 수 있다 → **repo를 public 전환하기 전에 이 폴더를 훑고,
  회사명이 든 자소서 최종본은 로컬/비공개로 옮긴다.** 소재은행·블로그 초안은 공개해도 무방.

## 티스토리 발행 절차

1. `blog/` 초안을 티스토리 에디터에 붙여넣기 (마크다운 모드)
2. 코드 블록/이미지 깨짐 확인 → 발행
3. 발행 URL을 글 상단 frontmatter의 `published:` 에 기록 (어디까지 발행했는지 추적)
