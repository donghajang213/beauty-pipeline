# 자소서 소재은행

> **사용법**: 프로젝트에서 "사건"이 생길 때마다 STAR 형식으로 즉시 기록한다.
> 자소서를 쓸 때는 문항에 맞는 소재를 골라 분량에 맞게 풀어쓰면 된다.
> STAR = Situation(상황) / Task(과제) / Action(행동) / Result(결과) — 국내 대기업 인성·경험 문항의 표준 구조.

## 양식 (복사용)

```
### 소재 N: (한 줄 제목)
- 어울리는 문항: (예: 문제해결 경험 / 꼼꼼함 / 학습 능력)
- S (상황):
- T (과제):
- A (행동):
- R (결과/배움):
- 증거: (커밋/PR/블로그 링크)
```

---

### 소재 1: 파이프라인보다 예산 알림을 먼저 만든 결정
- 어울리는 문항: 리스크 관리 / 우선순위 판단 / 주도성
- S: 한정된 GCP 크레딧(₩43만, 2개월)으로 포트폴리오 프로젝트를 시작해야 했다.
- T: 크레딧 소진 사고 없이 실습과 운영을 병행할 체계가 필요했다.
- A: 리소스 생성 전에 예산 알림부터 설정했다. 이때 기본 설정이 "크레딧 차감 후 청구액" 기준이라 크레딧이 소진될 때까지 알림이 오지 않는다는 맹점을 발견하고, gross(크레딧 제외) 기준으로 변경했다. 잔액을 월 단위 알림선(₩216,000)과 실제 목표 지출(월 2~5만)로 이원화해 고비용 기술(GKE 등) 실습 버퍼를 확보했다.
- R: 첫 주 지출 ₩0으로 인프라 구축 완료. "감시 장치는 기본값이 내 의도와 같은지부터 검증한다"는 원칙을 얻음.
- 증거: docs/setup/00-bootstrap.md, PLAN.md 6장, 블로그 001

### 소재 2: 시크릿 커밋 사고를 커밋 전에 차단
- 어울리는 문항: 꼼꼼함 / 보안 의식 / 실수로부터의 학습
- S: 공공데이터포털 인증키를 설정 파일에 넣는 과정에서, 커밋 대상인 예시 파일(.env.example)에 실제 키가 들어갔다.
- T: 키가 GitHub에 올라가기 전에 발견·차단하고, 재발을 막아야 했다.
- A: 커밋 전 점검에서 발견해 키를 비커밋 파일(.env)로 이전했다. 원인을 파보니 .gitignore의 인라인 주석이 패턴을 깨뜨리는 문제와 `.env.*` 패턴이 예시 파일까지 무시하는 문제가 겹쳐 있었다. `git check-ignore -v`로 규칙을 검증하는 절차를 도입하고, 예시 파일에 경고 주석을 명문화했다. (이후 CI에 gitleaks 시크릿 스캔 추가 예정)
- R: 유출 0건. "사람의 주의력이 아니라 장치(스캔, 검증 명령)로 막는다"는 DevSecOps 원칙을 체득.
- 증거: 커밋 5786730, RULES.md 4.1

### 소재 3: 크롤링 대신 합성 이벤트 생성기를 선택한 결정
- 어울리는 문항: 윤리/준법 의식 / 창의적 문제해결 / 기술 선택 근거
- S: 화장품 추천에 필요한 사용자 반응 데이터의 현실적 출처가 리뷰 사이트 크롤링뿐이었다.
- T: 약관 위반 소지 없이 스트리밍 파이프라인 역량을 증명할 데이터가 필요했다.
- A: 크롤링을 배제하고, 피부타입별 사용자 프로필이 조회/찜/구매 이벤트를 발생시키는 합성 이벤트 생성기를 설계했다. 실제 기업의 클릭스트림 파이프라인과 동일한 구조(이벤트 스키마 → Pub/Sub → 스트리밍 적재)를 갖춘다.
- R: 법적 리스크 제거 + 이벤트 스키마 설계라는 실무형 과제를 확보. 의사결정 과정을 ADR로 문서화.
- 증거: PLAN.md 2장

### 소재 4: 문서 없는 공공 API의 호출 경로를 실험으로 규명
- 어울리는 문항: 문제해결 / 집요함 / 학습 능력
- S: 승인받은 식약처 API 3종이 안내된 엔드포인트로 호출하면 500 오류만 반환했다.
- T: 상세 문서 없이 실제 호출 가능한 경로를 찾아야 했다.
- A: 공공 API의 명명 관례를 가설로 세우고, 후보 오퍼레이션 경로를 자동 순회하는 탐침 스크립트를 작성해 상태코드와 응답을 비교했다. 3개 API 모두 서로 다른 명명 패턴을 쓴다는 것을 확인하고 경로를 전부 찾아냈다.
- R: 총 61,000여 건의 실데이터 접근 확보. "외부 원천은 문서가 아니라 실호출로 검증한다"는 수집 계층 원칙 수립.
- 증거: ingestion/probe.py, 블로그 001
