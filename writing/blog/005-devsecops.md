# 파이프라인에 자물쇠 채우기 — 보안 게이트 3종, 키 없는 배포, 그리고 사람보다 빠른 알림

## 배경

[4편](004 링크)까지로 데이터는 흐르고 API는 서빙 중이다. Phase 3의 질문은 하나였다: **"이걸 안심하고 운영할 수 있는가?"** 답을 네 조각으로 만들었다 — 보안 게이트, 시크릿 금고, 알림, 자동 배포.

## 1. CI 보안 게이트 3종 — 층마다 다른 자물쇠

| 도구 | 지키는 층 | 우리 결과 |
|---|---|---|
| gitleaks | git 이력 속 시크릿 (과거 커밋까지) | 전체 이력 유출 0건 |
| Bandit | 파이썬 코드의 위험 패턴 | 오탐 3건 판단 후 근거와 함께 제외 |
| Trivy | 컨테이너 이미지의 CVE | HIGH/CRITICAL 0건 |

배운 것: **보안 도구의 경고는 "무조건 고침"이 아니라 "판단"이다.** Bandit이 SQL 조립이라고 지적한 3곳은 검토 결과 사용자 입력이 전부 파라미터 바인딩이었고(오탐), 제외하되 그 근거를 설정 파일에 남겼다. 반대로 Trivy 0건은 우연이 아니다 — slim 베이스 + non-root + 최소 의존성으로 이미지를 만든 설계의 결과다.

재미있는 순간: 도입 직후 Bandit이 시크릿 **이름**(`data-go-kr-service-key`)을 하드코딩된 비밀번호로 오인했다. 값이 아니라 금고 칸의 라벨인데 말이다. 새 게이트가 진짜로 물고 늘어진다는 증거였다.

## 2. Secret Manager — 그릇은 코드로, 값은 절대 코드에 넣지 않는다

인증키를 `.env`(내 PC)에서 클라우드 금고로 옮겼다. 여기서 원칙 하나:

> 시크릿의 "그릇"은 Terraform으로 만들지만, "값"은 절대 Terraform에 넣지 않는다.
> 코드에 쓰면 git 유출이고, 참조만 해도 state 파일에 평문으로 남는다.

코드는 `환경변수 → Secret Manager → 명확한 에러` 순서로 키를 찾는다. 로컬은 .env로 간편하게, 클라우드는 파일 없이 IAM 권한만으로.

이 작업이 당일에 바로 보답했다: `.env`를 파싱하다가 **키가 편집 실수로 주석 줄에 흡수되어 깨져 있는 걸 발견**한 것. 그대로면 다음 새벽 배치가 인증 실패로 죽었을 텐데, 이제는 .env가 깨져도 파이프라인이 금고에서 키를 꺼내 계속 돈다.

## 3. 알림 — 받으면 행동할 것에만

- **API 다운**: 전 세계에서 5분마다 /health 호출(uptime check), 5분 지속 실패 시 메일. 콜드스타트의 순간 실패 1번은 무시한다 — 소음 많은 알림은 곧 무시되는 알림이니까.
- **API 5xx**: 정상 상태에선 0이어야 하므로 1건도 알림.
- **비용**: Phase 0의 예산 알림이 담당.
- **DAG 실패**: 미구현. Airflow가 로컬 Docker라 클라우드 모니터링이 볼 수 없다 — 로컬 실행(비용 0원)을 선택한 트레이드오프로 정직하게 문서화했다. 한계를 아는 것도 설계의 일부다.

## 4. CD — 훔칠 키가 없는 배포

main에 API 코드가 병합되면 GitHub Actions가 빌드→푸시→배포→스모크 테스트까지 자동으로 한다. 핵심은 인증 방식:

> **Workload Identity Federation**: 서비스 계정 키를 GitHub에 저장하는 대신,
> GitHub가 워크플로마다 발급하는 단기 OIDC 토큰을 GCP가 검증하고
> "우리 저장소에서 온 요청"에만 배포 권한을 잠깐 빌려준다. **훔칠 키 자체가 없다.**

디테일 두 개: 이미지 태그는 커밋 SHA(지금 서빙 중인 코드가 어느 커밋인지 즉시 추적), 그리고 Terraform에는 "이미지는 CD 소유물"이라고 선언(lifecycle ignore_changes)해서 두 도구가 서로의 변경을 되돌리며 싸우지 않게 했다.

첫 자동 배포는 수동 트리거로 실증했다 — 전 단계 초록, 새 리비전이 트래픽 100%.

## 배운 것

- 보안은 기능이 아니라 층층이 쌓는 습관: 이력(gitleaks) / 코드(Bandit) / 이미지(Trivy) / 자격증명(WIF·SM) / 권한(최소 IAM)
- 모든 도구 도입에는 "끈 경고의 근거"가 남아야 한다 — 그게 감사(audit) 가능성이다
